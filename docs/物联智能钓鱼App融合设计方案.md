# 物联智能钓鱼 App 融合设计方案

生成日期：2026-06-12

## 1. 融合目标

本版本以“物联智能设备驱动钓鱼决策”为核心，把 v1 的设备驾驶舱能力和当前项目的 AI 出钓建议、钓点生态、鱼获记录、鱼圈社区、商城服务、个人中心整合为一个完整 App。

产品主线从“今天能不能钓”升级为：

1. 设备先感知：智能浮漂、探鱼器、钓箱、钓伞、钓台持续采集水情、鱼口、安全和装备状态。
2. AI 再判断：结合天气、钓点、近期鱼获和设备遥测生成今日策略。
3. 用户去执行：导航、现场修正、装备核对、鱼获记录形成闭环。
4. 数据再回流：鱼获、设备状态和用户复盘回到鱼情模型中。

## 2. v1 与当前项目的融合点

| 来源 | 可继承能力 | 融合后的定位 |
| --- | --- | --- |
| v1 首页 IoT 看板 | 水温、PH、溶氧、风速、设备列表、开关状态 | 升级为首页“物联设备中枢”，成为 App 第一心智 |
| v1 智能设备 Provider | 本地设备状态、开关控制、设备摘要 | 迁移到当前项目 `iot_device_provider.dart`，后续可替换为真实设备 API |
| v1 商城 / 发布 / 我的 | 高保真入口和移动端交互 | 当前项目已具备更完整闭环，保留当前版本为主 |
| 当前首页 AI 决策 | 推荐摘要、钓法计划、现场修正、推荐依据 | 增加设备遥测权重，让 AI 结论更可信 |
| 当前钓点 / 鱼获 / 鱼圈 / 商城 | 钓点详情、鱼获记录、社区流、交易服务 | 作为设备感知后的执行和沉淀链路 |
| 当前水墨主题 | 国风纸纹、湖色、毛玻璃卡片、书法标题 | 继续保留，但加入“智能仪表盘”信息密度 |

## 3. 信息架构

底部主导航建议为：

| 入口 | 页面定位 |
| --- | --- |
| 设备 | 物联设备中枢 + AI 今日出钓策略 |
| 钓点 | 钓点生态、地图图层、推荐钓场、钓点详情 |
| 发布 | 鱼获记录、现场复盘、照片标签、自动文案 |
| 鱼圈 | 鱼获动态、钓友经验、后续点赞评论 |
| 我的 | 个人资料、设置、运营监控、服务和权益 |

商城不放入主 Tab，而是作为“装备清单、AI 配装、我的服务”中的高频入口进入，避免主导航过重。

## 4. 核心页面效果

### 4.1 设备首页

首屏结构：

1. 品牌和定位状态：显示当前钓点、GPS/默认钓点状态、智能提醒。
2. AI 出钓结论大卡：展示分数、目标鱼、最佳窗口、站位、避坑和主行动。
3. 物联设备中枢：展示在线设备数、最低电量、平均信号、核心设备结论。
4. 横向设备卡：智能浮漂、探鱼器、钓箱、钓伞、钓台可横滑查看并本地切换运行/待机。
5. 今日计划：把设备结论落到具体打法、装备、止损规则。
6. 一键行动：导航、现场修正、装备清单、记录鱼获。

视觉关键词：

- 水墨纸纹背景保留，避免过科技导致与“江湖钓客”割裂。
- 设备中枢使用深湖绿到湖蓝渐变，强调“智能核心”。
- 设备状态用小型仪表、进度条、运行标签表达，信息密度高但不压迫。
- 告警类状态使用朱砂色，普通提醒使用芦苇金，稳定状态使用松绿色。

### 4.2 钓点页

钓点页继续承担“去哪里”的问题：

- 地图图层保留鱼情、钓场、活动、设备。
- 设备图层应展示附近可用智能设备、设备信号覆盖、传感器更新时间。
- 钓点详情增加“设备可信度”：例如探鱼器覆盖、水位传感器、夜钓安全设备。

### 4.3 鱼获记录页

鱼获记录从“手动发帖”升级为“设备辅助复盘”：

- 自动带入钓点、水温、水深、鱼层、天气、设备状态。
- 鱼获照片、鱼种、尺寸、重量仍由用户确认。
- 发布前展示设备辅助生成的复盘文案。

### 4.4 鱼圈社区

社区内容应突出真实数据价值：

- 鱼获动态标签显示“设备验证”“水情同步”“低概率战绩”。
- 后续可按设备类型筛选，例如“浮漂实测”“探鱼器鱼层”“夜钓安全”。

### 4.5 商城服务

商城重点从普通装备售卖调整为“智能设备 + 钓法配装”：

- 推荐智能浮漂、探鱼器、钓箱传感模块、钓伞联动模块。
- 与首页装备清单联动，一键加入购物车。
- 提供设备校准、售后、租赁和会员服务。

## 5. 当前已落地改动

- 首页辅助信号改为读取同一套设备状态。
- 设备详情弹窗可展示设备清单。

## 6. 后续优先级

1. 真实 IoT 接入：设备绑定、MQTT/WebSocket 或 HTTP 轮询、设备在线状态、指令下发。
2. 设备数据入库：保存遥测快照、鱼口事件、水温水深、信号质量和校准记录。
3. AI 结果加权：后端推荐接口接收设备快照，输出设备参与推荐的理由。
4. 设备详情页：从首页弹窗升级为独立页面，支持历史曲线、告警、固件、共享权限。
5. 设计图资产落地：现有 `assets/images/design/` 已包含页面效果设计图，可继续作为页面视觉验收参考。
6. 商城设备闭环：智能设备商品、租赁、绑定、售后和保养记录打通。
