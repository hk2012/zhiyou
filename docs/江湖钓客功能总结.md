# 江湖钓客 App 功能总结

生成日期：2026-06-10

用途：本文档可作为 ChatGPT 网页版项目的项目资料上传，用于让后续对话快速理解当前 app 的产品定位、功能范围、技术架构和后续规划。

## 1. 产品定位

江湖钓客是一款面向休闲垂钓用户的移动端应用，核心目标是把“今天去哪里钓、怎么钓、是否安全、钓完如何记录和分享”串成完整闭环。

当前版本采用水墨国风视觉风格，前端使用 Flutter 构建，后端使用 FastAPI 提供首页推荐、AI 垂钓分析、钓点生态、鱼获记录、共享商城、账号和运营监控能力。

## 2. 核心用户场景

1. 出钓前判断：根据位置、天气、水情、目标鱼和历史规则生成今日出钓建议。
2. 现场修正：用户补充水色、风口、人流、鱼口等现场观察后，重新生成更贴近现场的方案。
3. 找钓点：查看钓点生态图层、推荐钓场、钓点详情、安全风险和可用服务。
4. 记录鱼获：保存鱼种、钓法、水域、尺寸、重量、照片标签、笔记、发布范围等鱼获档案。
5. 社区分享：把鱼获记录映射为鱼圈动态，形成内容流和后续互动基础。
6. 装备与服务交易：围绕今日鱼情推荐装备组合，并提供购物车、结算、租赁、售后、会员权益等服务市场入口。
7. 个人中心与设置：展示用户资料、统计、运营状态、账号安全、隐私、通知、位置服务和帮助反馈。

## 3. 前端功能模块

### 3.1 登录与账号

- 通过手机号和密码登录，当前后端使用本地 demo 账号用于联调。
- 登录成功后保存 token 和用户信息，主页面通过路由守卫保护。
- 支持退出登录、切换账号入口，以及个人资料、兴趣、统计信息展示。

### 3.2 首页

首页是产品的核心决策页，围绕“今天能不能钓、该怎么钓”展开。

- 读取定位信息，默认可回退到预设钓点。
- 请求后端生成首页推荐摘要。
- 展示综合评分、结论、最佳窗口、站位提示、线组或装备提示。
- 展示目标鱼推荐、钓法匹配、避坑建议、低概率挑战、季节水情建议、安全风险、数据源可信度、近期有效打法等卡片。
- 支持首页卡片显示配置和排序偏好保存。
- 支持“现场修正”：将用户现场观察发送给后端重新分析。
- 支持 AI 垂钓分析：由后端代理 OpenAI 请求；没有 API Key 或网络异常时回退到本地规则。
- 支持低概率战绩记录，生成鼓励标题和分享文案。

### 3.3 钓点 / 发现页

发现页用于展示钓点生态和平台资源。

- 支持鱼情、钓场、活动、设备等图层配置。
- 展示宏观信号卡，例如鱼情、气象、资源连接等摘要。
- 展示地图标记，使用相对坐标渲染到水墨地图上。
- 展示推荐钓场，包括区域、等级、AI 评分、数据源、设备数和活跃度。
- 展示生态资源入口，例如钓友、商家、设备、活动等。
- 支持进入钓点详情页。

### 3.4 钓点详情

钓点详情页用于把一个具体钓点的决策信息展开。

- 展示钓点名称、头图文案、评分、距离、地址提示、最佳窗口、水温、水深、鱼活性、风险等级、停车和路程时间。
- 展示目标鱼、活性、推荐钓法和窗口。
- 展示设施、规则、战术、安全清单、服务、预测等结构化内容。
- 支持收藏钓点，并可保存备注。
- 支持从钓点详情进入鱼获记录，自动带入钓点名称。

### 3.5 发布 / 鱼获记录

发布入口位于底部导航中心按钮，是鱼获记录和社区内容的来源。

- 分步骤录入关键事实：水域、鱼种、尺寸、重量。
- 录入复盘信息：钓法、水色、鱼口、设备状态。
- 支持现场拍照或从相册选择，当前以照片标签和占位交互为主。
- 支持自动生成标题和分享文案。
- 支持自动排版预览。
- 支持草稿保存或发布动态。
- 支持发布范围设置。

### 3.6 鱼圈 / 社区

鱼圈当前通过鱼获记录列表生成动态流。

- 从 `/records/catches` 读取鱼获记录。
- 将鱼获记录映射为作者、钓点、标签、内容、点赞数和评论数。
- 提供动态列表、空状态、操作面板等社区基础交互。
- 为后续点赞、评论、关注、推荐流和隐私控制预留结构。

### 3.7 商城服务

商城服务页已经具备较完整的交易体验原型。

- 展示服务分类：装备、租赁、向导、设备、售后等。
- 展示商城轮播、服务项、信任保障和认证合作服务商。
- 支持商品搜索、分类筛选、全部商品面板。
- 支持 AI 装备组合，一键加入购物车。
- 支持收藏商品、咨询服务、购物车数量管理。
- 支持优惠券、积分抵扣、会员折扣、配送方式选择和确认结算。
- 支持订单、租赁预约、售后管家、维修保养、押金退还等入口。

### 3.8 我的 / 设置 / 运营

个人中心承担用户资产、产品状态和设置入口。

- 展示用户资料、等级、兴趣和统计数据。
- 提供进入商城、设置、运营监控等入口。
- 运营监控展示产品模块、架构层、能力分组、服务健康、请求统计和近期请求。
- 设置页包含账号与安全、隐私设置、实名认证、消息通知、鱼情提醒、位置服务、动态天气主题、缓存、语言、字体、关于、协议、隐私政策、帮助反馈等内容。

## 4. 后端能力

后端统一挂载在 `/api/v1`，主要模块如下。

| 模块 | 接口前缀 | 主要能力 |
| --- | --- | --- |
| 健康检查 | `/health` | 存活检查、就绪检查、数据库连通性 |
| 账号认证 | `/auth` | demo 密码登录、退出登录 |
| 用户中心 | `/user` | 当前用户资料、资料更新、用户统计 |
| 首页推荐 | `/home` | 首页卡片配置、推荐摘要、现场二次分析、低概率战绩、数据库预览 |
| AI 垂钓分析 | `/ai` | 后端代理 OpenAI Responses API，失败时本地规则兜底 |
| 发现生态 | `/explore` | 发现页总览、钓点详情、按名称查钓点、收藏钓点 |
| 共享商城 | `/mall` | 商城首页总览、分类、轮播、服务、保障、合作商 |
| 鱼获记录 | `/records` | 创建鱼获记录、读取鱼获列表 |
| 运营监控 | `/ops` | 产品总览、运行监控、系统事件、请求日志 |

## 5. 数据模型概览

当前数据库模型覆盖以下核心实体：

- 用户与偏好：用户、首页卡片定义、用户首页卡片偏好。
- 钓点与水情：钓点、天气水情快照、钓点详情、钓点收藏。
- 推荐规则：玩法鱼种匹配、近期有效打法、季节水情建议、安全风险规则、数据源状态、推荐运行记录、现场观察记录。
- 鱼获与内容：低概率战绩、完整鱼获日志、照片标签、自动排版 JSON、发布状态和可见范围。
- 发现生态：发现页图层、信号卡、地图标记、推荐钓场、生态资源项。
- 商城服务：服务分类、轮播、服务卡片、信任保障、认证合作商。
- 运营监控：产品模块、系统事件、请求日志。

本地开发默认使用 SQLite，部署时可通过环境变量切换 PostgreSQL。后端配置中也预留了 Redis、OpenAI API Key 和 OpenAI 模型名。

## 6. 技术架构

### 前端

- Flutter + Riverpod + GoRouter。
- Dio 作为 HTTP 客户端，并统一注入 Authorization token。
- ScreenUtil 做移动端设计尺寸适配。
- 宽屏 Web 环境下将 app 内容限制为手机宽度，并使用水墨主题背景居中展示。
- iOS、Web 目录已存在，当前实现偏移动端体验。

### 后端

- FastAPI + SQLAlchemy + Pydantic。
- 中间件记录请求 ID、耗时、状态码和最近请求，用于运营监控。
- CORS 通过环境变量配置。
- OpenAI API Key 只保存在后端环境变量中，前端不直接接触密钥。
- AI 分析使用 OpenAI Responses API；异常或未配置密钥时返回本地规则分析。

## 7. 当前实现状态

已实现或已有可运行骨架：

- 登录、路由守卫、主导航。
- 首页推荐、现场修正、AI 分析兜底、首页卡片偏好。
- 发现页、钓点详情、收藏钓点。
- 鱼获记录创建与列表读取。
- 鱼圈动态基础列表。
- 商城交易体验原型，包括购物车和结算闭环。
- 用户中心、设置页、运营监控。
- 后端接口、数据库模型、种子数据和接口测试。

仍偏原型或需要生产化：

- 账号体系目前是 demo 登录，尚未接入真实用户表、密码哈希、验证码、刷新令牌和权限。
- 社区互动的点赞、评论、关注、推荐流还未形成完整后端闭环。
- 商城订单、支付、库存、履约、售后当前主要是前端体验原型。
- 地图能力使用静态水墨图和相对标记，尚未接入真实地图 SDK 或地理围栏。
- 图片上传、媒体存储、内容审核和隐私脱敏需要补齐。
- OpenAI 分析需要生产环境配置 API Key、限流、审计和成本控制。
- 运营日志目前包含内存态请求记录，生产环境需要持久化和告警。

## 8. 后续建议

优先级建议：

1. 完成真实账号体系：验证码、密码哈希、token 刷新、用户表、权限和设备管理。
2. 打通鱼获闭环：图片上传、鱼获详情、编辑、删除、发布状态、隐私范围、社区展示。
3. 强化钓点能力：真实定位、钓点列表、收藏列表、距离排序、钓点隐私脱敏、违规钓点提醒。
4. 生产化 AI 分析：API Key 管理、调用日志、失败重试、费用监控、安全提示模板和结果缓存。
5. 商城后端化：商品表、订单表、支付状态、库存、租赁时段、售后工单。
6. 运营后台：模块开关、种子数据管理、系统事件、请求日志、数据源健康和告警。
7. 测试与发布：补充 Flutter Widget / Golden 测试、后端集成测试、CI、环境变量模板和部署文档。
